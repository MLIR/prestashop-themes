Introduzione

Cos'è PrestaShop e-commerce?

PrestaShop è una soluzione e-commerce open source e gratuita (con il kernel di base). Supporta vari canali di pagamento come Google Checkout, PayPal o pagamenti via API.

PrestaShop è disponibile su Licenza Open Software ed è stato ufficialmente introdotto nel 2007, pensato per aziende di medie o piccole dimensioni. Il software, basato sul motore Smarty template, è ormai utilizzato da migliaia di negozi in tutto il mondo.

Cos'è PrestaShop template?

PrestaShop template è un tema per la soluzione e-commerce PrestaShop. Offre all'utente ampie possibilità di personalizzazione e quindi di creare uno store online che risponda davvero alle sue esigenze.

Struttura dei Files

Il pacchetto del template che hai scaricato si suddivide in diverse cartelle. Vediamo cosa contiene ciascuna di esse :

  • screenshots - contiene schermate del template. Non sono file per la produzione
  • sources.zip - contiene i file sorgente del template
    • sources
      • psd - file sorgente Adobe Photoshop (.psd)
  • theme - contiene i file del tema PrestaShop
    • manual_install - contiene i file per l'installazione manuale
      • img - imaggini per l'installazione manuale
      • modules - moduli per l'installazione manuale
      • themes - file del tema per l'installazione manuale
      • dump.sql - dati campione per l'installazione manuale
    • sample_data - contiene file utilizzati per l'installaziode dei dati campione con modulo sampledatainstall
    • themeinstallator - contiene un archivio per l'installazione automatica del tema
      • theme####.zip - archivio tema. Contiene tutti i file del tema. È installato usando il metodo di installazione standard di PrestaShop
  • fonts_info.txt - contiene link da cui scaricare i font custom del tema
  • info.txt - contiene istruzioni su come estrarre i file sorgente

Per iniziare

Preparativi

Prima di iniziare l'installazione di PrestaShop e-commerce devi assicurarti di avere tutto l'occorrente. I preparativi da completare sono i seguenti:

Software per l'Editing

Per poter lavorare sul template PrestaShop con la massima comodità, ti consigliamo di scaricare tutte le applicazioni necessarie. La lista del software necessario è consultabile sulla pagina di anteprima del template. I requisiti possono variare a seconda del template, ma in generale ciò che servirà dovrebbe essere quanto segue:

  1. Prima di tutto devi avere un'applicazione per estrarre l'archivio sources_#########.zip, protetto da password. Si può usare WinZip 9+ (per Windows) o Stuffit Expander 10+ (per Mac).
  2. Potresti anche aver bisogno di Adobe Photoshop, da utlizzare per modificare i file sorgente .PSD o per modificare immagini e design del template, se lo desideri.
  3. Per modificare i file sorgente del template avrai bisogno di un editor di codice, come Adobe Dreamweaver, Notepad++, Sublime Text, ecc.
  4. Per caricare i file sul server potresti aver bisogno di un gestore FTP Manager, come Total Commander, FileZilla, CuteFTP, ecc.

Requisiti del Server

Verifica che il tuo server risponda ai requisiti minimi di PrestaShop:

  1. Sistema: Unix, Linux o Windows. Si consiglia vivamente Unix.
  2. Web server: Apache Web server 1.3 o successivi, Microsoft IIS Web server 6.0 o successivi, e nginx 1.0 o successivi.
  3. PHP 5.2* o successivi.
  4. MySQL 5.0 o successivi.
  5. Almeno 32 Mb di RAM sul server (64 Mb è vanno bene, ma più ce n'è, meglio è).
PrestaShop funziona con PHP 5.1 o successivi, ma le versioni inferiori alla 5.2 hanno dei bug che potrebbero impedire il corretto funzionamento di alcune funzioni (ad esempio problemi di orari e fusi orari).

Usare il server locale

Puoi eseguire PrestaShop localmente sul tuo computer usando il server locale. Per creare un server locale, utilizza applicazioni localhost, come WAMP, AppServ, MAMP, ecc. Sono tutti programmi normalmente installabili che potrai utilizzare per eseguire PrestaShop.

Guarda i seguenti video per ulteriori istruzioni su come configurare un ambiente di sviluppo locale:

  • Come installare un ambiente di sviluppo AppServ
  • Come installare un ambiente di sviluppo WAMP
  • Come installare un ambiente di sviluppo XAMP

Scaricare

Per installare PrestaShop devi scaricare il pacchetto del motore PrestaShop dal sito ufficiale all'indirizzo www.prestashop.com/en/downloads

Basta inserire il tuo nome e indirizzo e-mail nella casella Download e poi cliccare su 'Download'.

Verifica che la tua versione del motore PrestaShop corrisponda ai requisiti del template. Tutte le versioni di PrestaShop sono disponibili su SourceForge.

Estrarre i File

Una volta scaricato il pacchetto PrestaShop dovrai estrarre file e cartelle dall'archivio.

È possibile farlo con qualsiasi applicazione per la gestione di archivi come WinZip(PC), StuffitExpander(MAC), ecc.

Per ulteriori istruzioni su come estrarre i file dagli archivi puoi guardare questi tutorial: WinZip(PC) e StuffitExpander(MAC).

Nota: se il tuo pannello di controllo di hosting ha l'opzione 'Estrai', puoi utilizzarla per estrarre i file direttamente sul tuo server subito dopo aver caricato il file .zip.

Caricare i file del motore PrestaShop

Se hai estatto i file localmente sul tuo computer dovrai ora caricarli sul tuo server.

Puoi farlo utilizzando il File Manager del pannello di controllo del tuo servizio hosting oppure utilizzando altre applicazioni per gestione FTP come FileZilla, TotalCommander, CuteFTP, ecc.

Qui ci sono anche delle guide video più dettagliate su come caricare i file sul server usando un gestore FTP e come caricare i file sul server usando cPanel (WebHost Manager, WHM).

Installazione motore PrestaShop

Una volta terminato il caricamento dei file potrai avviare l'installazione del motore PrestaShop. Apri il browser e digita il nome di dominio/percorso directory PrestaShop nella barra degli indirizzi e poi aggiorna la pagina. Adesso dovresti vedere la schermata di benvenuto. Per installare PrestaShop, segui queste istruzioni:

Fase 1: Assistente Installazione (Seleziona la lingua)

Qui puoi selezionare la lingua utilizzata per l'installazione

Nota: la lingua scelta per l'installazione non determina la lingua per il negozio vero e proprio. La lingua predefinita di PrestaShop è l'inglese. Per cambiare la lingua si possono installare pacchetti di localizzazione aggiuntivi.

Una volta fatto, clicca su 'Next'.

Fase 2: Assistente Installazione (Accordo di Licenza)

Leggi l'accordo di licenza.

Una volta fatto, clicca su 'Next'.

Fase 3: Compatibilità Sistema

Verifica le impostazioni PHP e i permessi cartella. In caso di errori, contatta il tuo provider per chiedere assistenza.

Se tutto sembra a posto, clicca su 'Next'.

Fase 4: Configurazione Negozio

In questa fase dovrai inserire impostazioni di base del negozio, come nome, attività, paese, fuso orario, ecc.

Ognuna di queste impostazioni potrà anche essere modificata in seguito dal pannello di amministrazione di PrestaShop.

Fase 5: Configurazione Sistema

Se hai già un database MySQL, inserisci i dettagli del database nei rispettivi campi.

Altrimenti puoi trovare i dettagli del database sul pannello di controllo hosting oppure contattando il tuo provider.

Una volta finito, clicca su 'Next'.

Fase 6: Installazione completata

Questa è la fase finale dell'installazione.

Per ragioni di sicurezza dovrai eliminare la directory /install/ sul tuo server.
A partire dalle versioni 1.5.4, PrestaShop rinominerà automaticamente la cartella di amministrazione in "admin" seguito da 4 cifre a caso, ad es. "admin8814." L'operazione di riassegnazione del nome avviene appena accedi alla cartella "admin" per la prima volta: PrestaShop la rinomina e poi ti fa accedere correttamente alla cartella rinominata.

Accedi al back office di PrestaShop andando sulla cartella "admin" appena rinominata e inizia a inserire i tuoi dati nel catalogo.

Se incontri difficoltà puoi consultare il manuale ufficiale di installazione PrestaShop.

Installazione Template

1. Importa il tema

Nota: questo metodo non fornisce dati campioni per l'installazione.

1. Preparazione Template.

Prima di procedere verifica di aver già scaricato il template e di averne estratto i file. Puoi consultare le guide dettagliate su come estrarre i file del pacchetto template usando WinZip e StuffitExpander.

Una volta estratti i file, puoi procedere con il caricamento del template.

2. Installazione.

Accedi al pannello di amministrazione PrestaShop e vai su Preferences -> Themes. In alto sulla pagina, clicca su Add new theme.

Clicca su Add file nell'area "Import from your computer". Naviga sul pacchetto template estratto in precedenza e apri la cartella themeinstallator.

Seleziona il file .zip del tuo template e clicca su Save.

A questo punto inizierà la procedura di upload. Poi vedrai il messaggio che il tema è stato caricato con successo - "Successful upload". La nuova anteprima del tuo tema apparirà nella lista di temi installati.

Per attivare il tema, porta il puntatore sull'icona di anteprima del tema e clicca su Use this theme.

Adesso dovresti vedere la schermata con le informazioni sui moduli utilizzati nel tema, il numero di moduli attualmente attivati o disattivati. Clicca su Save per continuare.

A questo punto vedrai un messaggio che ti comunica della riuscita attivazione del tema: il tema è stato intallato correttamente e le immagini sono state rigenerate alle dimensioni definite. Per terminare l'installazione, clicca su Finish.

2. Installazione automatica con dati campione (installer 'Sample data')

Dopo aver installato i dati campione, il tuo negozio PrestaShop apparirà proprio come nella demo live del tuo template. Conterrà tutti i prodotti dimostrativi, le categorie, i moduli e le impostazioni dello store.

Attenzione a non utilizzare i dati campioni per il tuo negozio già esistente! Così facendo sostituiresti tutti i prodotti e le impostazioni del tuo negozio.

Per installare i dati campione usando il modulo Sample data installer, segui le istruzioni a partire dalla Fase 1 (Theme import) di questo manuale. Poi nel tuo pannello di amministrazione del negozio vedrai il modulo Sample data installer.

Usando questo modulo potrai facilmente installare i dati campione in pochi semplici passi. Fai come segue:

  1. Dal menu del pannello di amministrazione clicca sull'elemento "Install Demo Store".

  2. Seleziona e carica i file dei dati campione dalla directory 'sample_data' del tuo pacchetto template. Puoi farlo utilizzando il pulsante "Browse local files" oppure semplicemnte trascinando i file dati campione sulla finestra del browser.

  3. Attendi che i file vengano caricati.

  4. Poi clicca su "Continue Install".

  5. Adesso il Sample data installer configurerà automaticamente il tuo negozio e gli altri moduli. Per queste operazioni potrebbero volerci alcuni minuti.

  6. Una volta che i dati campione sono stati installati correttamente, dovrai completare un'ultima fase: rigenerare le imagini. Clicca su "Go To Regenerate Page".

  7. Per rigenerare le immagini, clicca su "Regenerate thumbnails".

Congratulazioni, il tuo store dimostrativo è ora installato.

3. Installazione manuale del template con dati campione

Dopo aver installato i dati campione, il tuo negozio PrestaShop avrà lo stesso aspetto del demo live. Conterrà tutti i prodotti dimostrativi, le categorie, i moduli, le impostazioni dello store, ecc.

Attenzione a non utilizzare i dati campioni per il tuo negozio già esistente! Così facendo sostituiresti tutti i prodotti e le impostazioni del tuo negozio.

1. Caricare i file del template

Per caricare il template sul tuo server:

  1. Apri il pacchetto del template.
  2. Vai sulla cartella /theme/manual_install/.
  3. Carica le cartelle /img/, /modules/ e /themes/ sulla directory root di PrestaShop.

Qui puoi seguire delle guide video dettagliate su come caricare i file sul server usando gestori FTP e come caricare i file sul server usando cPanel (WebHost Manager, WHM).

2. Installare i dati campione

I dati campione vengono forniti in forma di file .SQL. Il file si trova su theme/manual_install/ nel pacchetto template e si chiama dump.sql.

Il file dump.sql può essere installato utilizzando lo strumento phpMyAdmin o altri stumenti di gestione database presenti nel pannello di controllo del tuo servizio hosting. Questa guida approfondita su come installare file SQL potrà esserti utile.

Prima di procedere fai il back up del tuo database. Qui puoi vedere un video con istruzioni dettagliate su come fare il back up per un database.

Verifica che la tua versione del motore PrestaShop corrisponda ai requisiti del template. Altrimenti il database potrebbe essere danneggiato.

Una volta terminata l'installazione del file dump.sql dovrai attivare il template.

Nel pannello di amministrazione di PrestaShop, vai su Preferences > Themes, seleziona il tuo template nella sezione Themes e clicca su Save.

3. Aggiungere lingue al negozio online.

Dopo aver installato il template multilingua, sarai in grado di aggiungere tutte le lingue incluse (inglese, francese, spagnolo, tedesco, russo) sulla piattaforma del negozio online. Si può fare così:

Vai su 'Localization > Translations' nel pannello di amministrazione, trova 'ADD/UPDATE A LANGUAGE' e aggiungi le lingue che desideri selezionandole dal menù a tendina (1) e poi cliccando su 'Add or update a language' (2).

Se non vuoi aggiungere altre lingue, puoi eliminarle (2) oppure disabilitarle (1) sul menù 'Localization > Languages'.

4. Installare il template su di un negozio pre-esistente

Puoi usare questo metodo di installazione se hai già un sito PrestaShop attivo e vuoi soltanto installare un template nuovo.

1. Caricare i file del template e attivare il tema

  1. Apri il pacchetto template.
  2. Vai sulla cartella /theme/manual_install/.
  3. Carica le cartelle /modules/ e /themes/ sulla directory root di PrestaShop.

    Qui puoi vedere guide video dettagliate su come caricare file sul server utilizzando un gestore FTP e come caricare file sul server utilizzando cPanel (WebHost Manager, WHM).

  4. Dal tuo pannello di amministrazione PrestaShop vai su Preferences > Themes, seleziona il tema che hai installato (puoi anche usare il tema abilitato, non causerà alcun problema per il tema abilitato). Clicca su Advanced settings.

  5. Sulla schermata che apparirà, seleziona la directory con il tuo nuovo tema e clicca su Save.

2. Installazione Moduli

Ogni template include alcune impostazioni specifiche per i moduli. Per fare in modo che i moduli abbiano lo stesso aspetto dell'anteprima dimostrativa dovrai configurarli. Tutti i moduli possono essere configurati dal pannello di amministrazione di PrestaShop.

1. Dal tuo pannello di amministrazione PrestaShop, seleziona Modules > Modules dal menù principale.

Sulla pagina Modules potrai vedere la lista di moduli PrestaShop disponibili. Potrebbe essere necessario installarne alcuni.

Per installare un modulo, individualo sulla lista e clicca su 'Install'.

Usando i link del modulo potrai attivarli e disattivarli (Enable|Disable), riportare le impostazioni ai valori predefiniti (Reset), eliminare un modulo (Delete) oppure configurarlo (Configure).

2. Dopo aver attivato i moduli necessari, dovrai assegnarli alle posizioni corrette.

Nel pannello di amministrazione di PrestaShop, vai su Modules > Positions dal menù in alto. Per mettere un modulo nella posizione corretta, trascinalo nella posizione desiderata nella lista dei moduli.

3. Configurare le immagini

Su PrestaShop si possono usare dimensioni predefinite per le immagini dei prodotti, le immagini di categoria, i produttori, ecc.

Ad esempio, le immagini di dimensioni 'home' vengono solitamente utilizzate per le immagini prodotti negli elenchi di categoria, 'large' per le immagini principali del prodotto nella pagina principale del prodotto.

Per configurare le dimensioni di immagine del prodotto apri il pannello di amministrazione di PrestaShop, vai su Preferences > Images e imposta le idimensioni delle immagini come descritto su Configuration Page nella live demo del template.

Guide di Base

1. Come cambiare il nome del negozio

Per cambiare il titolo del tuo negozio apri il pannello di amministrazione di PrestaShop e vai su Preferences > Store Contacts, nella sezione Contact details.

Nel campo Shop name digita il nome del tuo negozio e clicca su Save per salvare le tue modifiche.

2. Come modificare le slide

Per modificare le immagini dello slider, trova il modulo Image slider for your homepage sulla pagina Modules del tuo negozio PrestaShop. Si trova nella sezione moduli Front Office Features. Clicca sul pulsante Configure per modificare il modulo.

Sulla pagina delle impostazioni dei moduli si possono cambiare le opzioni dello slider (come profondità, velocità, ecc.) e modificare le singole slide (immagine slide, URL, descrizioni, ecc.).

Moduli ed estensioni custom

Smart Blog

Smart blog - è una raccolta di moduli che ti permettono di creare un blog nel tuo negozio. I moduli sono i seguenti: Smart Blog, RSS Smart Blog Feed, Smart Blog Archive, Smart Blog Categories, Smart Blog Home Latest, Smart Blog Latest Comments, Smart Blog AddThis Module, Smart Blog Popular Posts, Smart Blog Recent Posts, Smart Blog Related Post, Smart Blog Search, Smart Blog Tags. Ha anche dei ganci - 'hooks' - dove i moduli del blog possono apparire sul sito, possono anche apparire negli hook standard (solo colonne).

Smart Blog

È il modulo principale, contenente diverse impostazioni sulle funzionalità del blog. Dopo l'installazione la scheda menù del Blog apparirà nel menù principale del tuo pannello di amministrazione, dove potrai aggiungere/rimuovere i post o categorie, o selezionare il tipo di immagine e le dimensioni da usare nel post (tipi e dimensioni delle immagini del post sono diverse dalle impostazioni delle immagini per lo store). Per aggiungere le impostazioni frontend del modulo, vai su Smart Blog (Modules->Front Office Features->Smart Blog) e aggiungi tutti i parametri.

Descrizione dei parametri:

  1. Meta Title - il titolo del tuo blog, che apparirà su tutte le pagine del blog.
  2. Meta Keyword - parole chiave che saranno usate dai motori di ricerca per l'ottimizzazione del SEO.
  3. Meta Description - descrizione del tuo blog utilizzato dai motori di ricerca per l'ottimizzazione del SEO.
  4. Main Blog Url - URL del tuo Blog.
  5. Use .html with Friendly Url - abilitare/disabilitare l'aggiunta di .html alle URL delle pagine del blog, usando le "Friendly Url".
  6. Number of posts per page - numero di post da mostrare su una pagina (categoria/pagina archivi, ecc.).
  7. Auto accepted comment - abilitare/disabilitare commenti degli utenti accettati automaticamente. Se abilitato, i commenti saranno disponibili senza moderazione dell'amministratore.
  8. Enable Captcha - abilitare/disabilitare il Captcha nel modulo per lasciare commenti su pagine dei prodotti.
  9. Enable Comment - abilitare/disabilitare lasciare commenti da parte degli utenti.
  10. Show Author Name - abilitare/disabilitare mostrare l'autore del post sulle pagine del blog
  11. Show Post Viewed - abilitare/disabilitare il numero di visualizzazione del post. Apparirà sotto il post sulle pagine del blog.
  12. Show Author Name Style - cambiare il formato del nome dell'autore.
  13. AVATAR Image: - aggiungere l'immagine avatar predefinita per tutti gli utenti.
  14. Show No Image - abilitare/disabilitare l'immagine. Si imposta No Image se non ci sono immagini per la categoria/post, etc.
  15. Show Category - abilitare/disabilitare immagine e descrizione della categoria sulla pagina delle categorie del blog.
  16. Blog Page Column Setting - impostare la visualizzazione dei moduli relativi al Blog. Questo richiede spesso impostazioni aggiuntive, per cui si consiglia di installare i moduli aggiuntivi nella stessa maniera in cui si intallano i moduli PrestaShop standard.
  17. Custom CSS - in questo campo si possono aggiungere stili aggiuntivi. Fai attenzione a utilizzare le classe già usate nel tema, altrimenti potrebbero verificarsi dei conflitti.
  18. Delete Old Thumblr - si usa per rigenerar le immagini blog nel caso in cui le dimensioni delle immagini venissero modificate o se si aggiungono nuovi tipi di immagine. Per rigenerare, seleziona Yes e clicca Regenerate.
SOLO ADESSO si possono installare altri moduli relativi al blog. La procedura di rimozione è l'opposto: PRIMA DEVI ELIMINARE TUTTI I MODULI RELATIVI e solo dopo elimina il modulo Smart Blog.

Modulo Media Parallax

Cos'è l'effetto parallasse?

Lo scorrimento parallasse è una speciale tecnica di scorrimento in cui le immagini nello sfondo si muovono più lentamente rispetto alle immagini in primo piano, creando così un illusione di profondità anche, ad esempio, in un videogioco 2D, per creare una sensazione di 'immersione'. Gli sviluppatori web iniziarono a introdurre l'effetto parallasse nel 2011, sfruttando l'HTML5 e il CSS3. Utilizzare il parallasse per i siti web è una tecnica di design sempre più popolare ed è visto come un modo semplice ma efficace per rappresentare la fluidità del web. Inoltre, i sostenitori della tecnica usano gli sfondi in parallasse come uno strumento per meglio coinvolgere l'utente e migliorare l'esperienza globale offerta dal sito web. Lo scorrimento in parallasse permette di creare un'esperienza di navigazione dinamica e interattiva, migliorando anche l'aspetto visivo.

Questo modulo permette di aggiungere un effetto parallasse al tuo sito web senza particolari sforzi. È possibile aggiungere in sfondo sia immagini che video.

Impostazioni: (solo per Google Chrome)

  1. Enable smooth scroll for Google Chrome browser - per abilitare un effetto aggiuntivo.
  2. Smooth scroll time - per impostare il tempo di scorrimento 'smooth'.
  3. Smooth scroll distance - per stabilira la distanza dell'effetto scorrimento 'smooth'.

Aggiungere il modulo Media parallax

  1. È necessario aggiungere l'id o la classe nel contenuto della tua pagina HTML. Si può anche usare un tag unico, che non deve essere ripetuto.

  3. Clicca su Add container in Add new block.

    Impostazioni:

    1. Selector - l'elemento a cui l'effetto verrà aggiunto (id o tag unico).
    2. Parallax type - il tipo di effetto che verrà utilizzato (immagine o video).
    3. Media files - per caricare tutti i file necessari.
    4. Clicca su Save.

  1. Nella lista di blocchi parallasse, individua il blocco desiderato e clicca su Delete.

Modulo Video Background

Questo modulo permette di aggiungere video a qualunque blocco sulla tua pagina HTML (deve esserci l'id o un tag unico).

Per aggiungere un video sullo sfondo segui questa procedura:

  1. Carica i file video e le immagini di anteprima sulla cartella modules/tmvideobackground/video (o su qualunque altra cartella sul server) via FTP. Il video va caricato in 3 formati: .mp4, .ogv, .webm, tutti i file con lo stesso nome (ocean.mp4, ocean.ogv, ocean.webm). Anche le immagini di anteprima vanno caricate in 3 formati: .jpg, .gif, .png e allo stesso modo deve avere lo stesso nome (ocean.jpg, ocean.gif, ocean.png).
  2. Dal pannello di amministrazione, apri il modulo Video background (Modules->Front Office Features->Video background module) e inserisci le impostazioni appropriate.

    Impostazioni:

    1. Video selector - l'elemento della pagina HTML in cui verrà aggiunto il video di sfondo(id o tag unico (body, header, footer, ecc.).
    2. Video path - il percorso verso il tuo file video (il percorso deve avere un simbolo / alla fine), i file possono essere messi su qualunque cartella del server ma il consiglio è di metterli in modules/tmvideobackground/video/.
    3. Video files name - i nomi dei tuoi file video. Va aggiunto solo il nome, senza aggiungere il formato (quindi, ad es., "ocean").
    4. Clicca su Save.
Per visualizzare il video, non va aggiunto nessun altro elemento di sottofondo per questo elemento.

Informativa sui Cookie

I cookie sono dei piccoli file di testo che un sito web crea sul computer di un visitatore. Vengono utilizzati tipicamente per identificare quello specifico visitatore allo scopo di fornirgli un'esperienza di navigazione migliore. I Cookie sono un elemento importantissimo per il corretto funzionamento di tante funzioni dei siti web che tutti ormai diamo per scontate.

Ad esempio, sono solitamente proprio i cookie che permettono ai negozi online di ricordare quali articoli hai messo nel tuo carrello. Sono usati anche per tenere attivo il tuo login sul sito, o per dare ai gestori del sito informazioni statistiche importanti.

Se gestisci un sito web, esso quasi sicuramente usa i cookie. La loro funzione più probabile è quella di monitorare il numero dei visitatori e i loro comportamenti, servendosi di strumenti come Google Analytics. Possono essere usati anche per mostrare agli utenti delle pubblicità il più possibile rilevanti, oppure - per chi vende prodotti - per far funzionare alcuni tra gli elementi più importanti del sistema di compravendita del tuo negozio online.

26 maggio 2012 - La ICO ha pubblicato una modifica all'applicazione della regolamentazione: "il consenso implicito" è ora consentito purché sia ben chiaro dove si trovano le tue informative su cookie e privacy.

Per la tua comodità abbiamo aggiunto un modulo Cookieslaw al nostro tema.

La configurazione è molto semplice.

  1. Lo trovi nella cartella manual_install che si trova nel pacchetto del template (se hai effettuato l'installazione manuale del template con dati campione, allora il modulo sarà aggiunto automaticamente).

  2. Poi, trovalo nella lista dei moduli nel tuo pannello di amministrazione (modulo Cookieslaw, tra i moduli "Front Office Features") e clicca su Install.

  3. Dopo l'installaziione (ignora questo passo se sei già stato automaticamente ridiretto sulla pagina di configurazione del modulo), clicca su Configure.

  4. Aggiungi tutte le impostazioni desiderate e clicca su Save.

    Impostazioni:
    • - Display at the top of the page(by default). Se deselezioni questa opzione il modulo verrà mostrato in fondo alla pagina.
    • - Link Cookies Policy.
    • - Link verso la pagina in cui l'utente viene ridiretto se non accetta le politiche sui cookie.

Caricare dati dai server Google

In alcuni paesi ci sono problemi a connettersi ai server di Google. Questo può provocare problemi nel caricamento dei font Google Fonts o delle librerie Java Script da Google CDN. Per risolvere questi problemi, provate a fare così:

Impossibile caricare i font di Google:

  1. Apri il file \themes\theme****\header.tpl.

  2. Individua la riga: 

    <link rel="stylesheet" href="http{if Tools::usingSecureMode()}s{/if}://fonts.googleapis.com/css?family=[---your_web_font_name_here---]" type="text/css" media="all" />

  3. Sostituiscila con la seguente: 

    <link rel="stylesheet" href="http://fonts.useso.com/css?family=[---your_web_font_name_here---]" type="text/css" media="all" />
