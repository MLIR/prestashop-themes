Introduction

Qu'est-ce que PrestaShop e-commerce?

PrestaShop est une solution d'e-commerce gratuite pour le noyau de base et open source. Elle prend en charge les plate-formes de paiement telles que Google checkout, paypal ou des paiements professionnels via des API.

PrestaShop est disponible sous Licence Open Software et a été lancée en Août 2007 pour les petites et moyennes entreprises. Le logiciel, qui est basé sur le moteur de template Smarty, est utilisé par des milliers de boutiques dans le monde.

Qu'est-ce que PrestaShop template?

PrestaShop template est un thème pour la solution d'e-commerce PrestaShop. Il propose une large variété d'options de personnalisation et permet de créer une boutique en ligne entièrement adaptée à vos besoins.

Structure des fichiers

Le package de template que vous avez téléchargé est composé de plusieurs dossiers. Voyons ce que contient chaque dossier:

  • screenshots - contient les captures d'écran du template. Pas pour la production
  • sources.zip - contient les fichiers source du template
    • sources
      • psd - fichiers source Adobe Photoshop (.psd) du template
  • theme - contient les fichiers du thème PrestaShop
    • manual_install - contient des fichiers pour l'installation manuelle
      • img - images pour l'installation manuelle
      • modules - modules pour l'installation manuelle
      • theme#### - fichiers du thème pour l'installation manuelle
      • dump.sql - exemple de données pour l'installation manuelle
    • sample_data - contient les fichiers utilisés pour l'installation des fichiers d'exemple avec le module sampledatainstall
    • themeinstallator - contient une archive pour l'installation automatique du thème
      • theme####.zip - archive du thème. Contient tous les fichiers du thème. Installation via la méthode d'installation standard de PrestaShop
  • fonts_info.txt - contient les liens où les polices personnalisées du template peuvent être téléchargées
  • info.txt - contient les instructions sur l'extraction des fichiers source

Démarrage rapide

Préparation

Avant de commencer à configurer votre PrestaShop e-commerce, assurez-vous que vous êtes bien préparé. Veuillez suivre les étapes de préparation suivantes:

Logiciel d'édition

Pour être à l'aise en travaillant avec votre template PrestaShop, nous vous recommandons de télécharger toutes les applications requises. Vous pouvez voir la liste des logiciels requis sur la page de prévisualisation du template. Les besoins peuvent varier selon le template, voici ce qui est généralement requis:

  1. Tout d'abord, il vous faut les bonnes applications pour extraire l'archive sources_#########.zip protégée par un mot de passe. Vous pouvez utiliser WinZip 9 et + (Windows) et Stuffit Expander 10 et + (Mac).
  2. Vous pouvez aussi avoir besoin de l'application Adobe Photoshop. Elle sert à modifier les fichiers source .PSD et elle est nécessaire si vous voulez modifier les graphiques et images du design du template.
  3. Pour modifier les fichiers du code source du template, il vous faut un éditeur de code comme Adobe Dreamweaver, Notepad++, Sublime Text, etc.
  4. Pour uploader les fichiers sur le serveur d'hébergement, vous pouvez avoir besoin du Gestionnaire FTP comme Total Commander, FileZilla, CuteFTP, etc.

Besoins côté serveur

Veuillez vous assurer que votre serveur d'hébergement est conforme aux besoins de PrestaShop:

  1. Système: Unix, Linux ou Windows. Unix est fortement recommandé.
  2. Serveur web: Serveur Web Apache 1.3 ou plus récent, serveur Web Microsoft IIS 6.0 ou plus récent, et nginx 1.0 ou plus récent.
  3. PHP 5.2* ou plus récent.
  4. MySQL 5.0 ou plus récent.
  5. Au moins 32 Mo de RAM sur votre serveur (64 Mb pour plus de confort. Plus il y en a, mieux c'est).
PrestaShop fonctionnera avec PHP 5.1 ou plus récent, mais les versions antérieures à la 5.2 présentent des bugs pouvant empêcher certaines fonctions d'agir convenablement (comme un fuseau horaire invalide).

Utilisation d'un serveur local

Vous pouvez lancer PrestaShop localement sur votre ordinateur à l'aide d'un serveur local. Pour créer un serveur d'hébergement local, veuillez utiliser les applications localhost telles que WAMP, AppServ, MAMP, etc. Toutes ces applications peuvent être facilement installées comme tout autre programme et utilisées pour lancer PrestaShop.

Veuillez consulter les tutoriels ci-dessous sur comment configurer un environnement de dévelopement local:

  • Comment installer l'environnement de développement web AppServ
  • Comment installer l'environnement de développement web WAMP
  • Comment installer l'environnement de développement web XAMP

Téléchargement

Pour installer PrestaShop, vous devrez télécharger le package de l'engine PrestaShop depuis le site officiel sur www.prestashop.com/en/downloads

Entrez juste votre nom et votre e-mail et cliquez sur le bouton 'Download'.

Assurez-vous que votre version de l'engine PrestaShop est conforme aux besoins du template. Toutes les versions de PrestaShop sont disponibles sur SourceForge.

Extraction des Fichiers

Une fois le package de l'engine PrestaShop téléchargé, vous devrez extraire ses fichiers et dossiers de l'archive.

Vous pouvez le faire avec n'importe quel gestionnaire d'archive comme WinZip(PC), StuffitExpander(MAC), etc.

Veuillez consulter les tutoriels suivants sur comment extraire les fichiers des archives avec WinZip(PC) et StuffitExpander(MAC).

NB: Si le panneau de contrôle de votre hébergeur propose l'option d''Extraire' les fichiers, vous pouvez l'utiliser pour extraire les fichiers directement sur votre serveur une fois le fichier .zip uploadé.

Upload des fichiers de l'engine PrestaShop

Si vous avez extrait les fichiers localement sur votre ordinateur, vous devrez les uploader sur votre serveur d'hébergement.

Vous pouvez le faire à l'aide du Gestionnaire de Fichiers du panneau de contrôle de votre hébergeur ou d'applications tierces de Gestion de FTP telles que FileZilla, TotalCommander, CuteFTP, etc.

Vous pouvez consulter le tutoriel vidéo détaillé sur comment uploader des fichiers sur un serveur à l'aide d'un gestionnaire FTP et comment uploader des fichiers sur un serveur à l'aide de cPanel (WebHost Manager, WHM).

Installation de l'engine PrestaShop

Une fois que vous avez fini avec l'upload des fichiers, vous pouvez passer à l'installation de l'engine PrestaShop. Ouvrez votre navigateur web et tapez 'votre nom de domaine/chemin vers votre répertoire PrestaShop' dans la barre d'adresse, puis rechargez la page. Vous devriez voir l'écran de bienvenue. Veuillez suivre les instructions ci-dessous pour installer PrestaShop.

Étape 1: Assistant d'Installation (Choisissez votre langue)

Choisissez la langue d'installation

NB: la langue d'installation n'affectera pas la langue de votre boutique. Par défaut, PrestaShop est en anglais. Vous pouvez installer des packages de localisation supplémentaires pour changer la langue de votre boutique.

Une fois que c'est fait, cliquez sur le bouton 'Next'.

Étape 2: Assistant d'Installation (Accords de Licence)

Veuillez accepter l'accord de licence.

Une fois que c'est fait, cliquez sur le bouton 'Next'.

Étape 3: Compatibilité du Système

Vérifiez les paramètres de PHP et les droits sur les répertoires. En cas d'erreur, veuillez contacter votre hébergeur pour être assisté.

Si tout est correct, cliquez sur le bouton 'Next'.

Étape 4: Configuration de la boutique

Cette étape nécessite que vous entriez les paramètres de votre boutique comme le nom, l'activité, le pays, le fuseau horaire, etc.

Tous ces paramètres peuvent être modifiés plus tard dans votre panneau d'administration PrestaShop.

Étape 5: Configuration du Système

Si vous disposez déjà d'une base de données MySQL, veuillez entrer les informations relatives à la base dans les champs appropriés ci-dessous.

Autrement, vous pouvez obtenir ces informations sur le panneau de contrôle de votre hébergeur ou en le contactant directement.

Une fois que c'est fait, cliquez sur le bouton 'Next'.

Étape 6: L'installation est terminée

Voici l'étape finale de l'installation.

Pour des raisons de sécurité, vous devez supprimer le répertoire /install/ de votre serveur d'hébergement.
À partir de la version 1.5.4, PrestaShop s'occupe du renommage du dossier d'administration pour vous: le nom devient le mot "admin" suivi de 4 chiffres aléatoires, par exemple "admin8814." Le renommage est fait dès que vous atteignez le dossier "admin" pour la première fois: PrestaShop le renomme puis accède correctement au dossier renommé.

Connectez-vous au back-office de PrestaShop en allant sur votre dossier "admin" nouvellement renommé et commencez à remplir votre catalogue avec vos données.

Si vous rencontrez des difficultés, veuillez consulter le manuel officiel d'installation de PrestaShop.

Installation du template

1. Importation du thème

NB: cette méthode ne propose pas d'installation de données d'exemple.

1. Préparation du template.

Avant de commencer, assurez-vous d'avoir téléchargé le template et extrait les fichiers du package du template. Vous pouvez consulter les tutoriels détaillés sur comment extraire le package du template à l'aide de WinZip et de StuffitExpander.

Une fois les fichiers extraits, vous pouvez passer à l'upload du template.

2. Installation.

Connectez-vous à votre panneau d'administration PrestaShop et allez dans Preferences -> Themes. En haut de la page cliquez sur le bouton Add new theme.

Cliquez sur le bouton Add file dans la section Import from your computer. Allez dans le package de template extrait auparavant et ouvrez le dossier themeinstallator.

Sélectionnez le fichier .zip de votre thème et cliquez sur Save.

Le processus d'upload va commencer. Vous verrez le message indiquant que le thème a été uploadé avec succès. L'aperçu de votre nouveau thème apparaîtra dans la liste des thèmes installés.

Pour activer le thème, placez le curseur sur l'icône d'aperçu du thème et cliquez sur Use this theme.

Vous devriez voir l'écran avec les informations sur les modules utilisés dans le thème, le nombre de modules activés et désactivés. Cliquez sur Save pour continuer.

Vous verrez ensuite la pop-up indiquant que l'activation du thème a réussi: le thème a été correctement installé, les images ont été générées à nouveau, conformément aux dimensions du thème définies. Pour terminer l'installation cliquez sur le bouton Finish.

2. Installation automatique avec des données de démo (Installeur de données d'exemple)

Après avoir installé les données d'exemple, votre boutique PrestaShop aura le même aspect que la live demo du template. Elle contiendra les produits de démo, les catégories, les modules et les paramètres de boutique.

N'utilisez pas les données sur votre site en production! Ceci remplacera tous les produits et paramètres de votre boutique.

Pour installer les données d'exemple avec le module d'installation de données d'exemple, veuillez suivre les instruction de l'étape 1 (Theme import) de ce manuel. Vous verrez ensuite le module d'installation de données d'exemple dans le panneau d'administration de votre boutique.

Avec ce module, vous pouvez facilement installer les données d'exemple en quelques étapes simples:

  1. Depuis le menu du panneau d'administration, cliquez sur l'élément "Install Demo Store".

  2. Sélectionnez et uploadez les fichiers des données d'exemple du dossier 'sample_data' du package de votre template. Vous pouvez le faire avec le bouton "Browse local files" ou en faisant glisser les fichiers de données d'exemple dans la fenêtre du navigateur.

  3. Attendez que les fichiers soient uploadés.

  4. Puis cliquez sur le bouton "Continue Install".

  5. L'installeur de données d'exemple configurera automatiquement votre boutique et les autres modules. Ceci peut prendre plusieurs minutes.

  6. Une fois les données d'exemple correctement installées, vous devez passer à la dernière étape - regénerer les images. Cliquez sur le bouton "Go To Regenerate Page".

  7. Pour régénérer les images, cliquez sur le bouton "Regenerate thumbnails".

Félicitations, votre boutique de démo est installée.

3. Installation manuelle du template avec des données d'exemple

Après l'installation des données d'exemple, vous verrez votre boutique PrestaShop apparaître comme dans le template de live demo. Elle contiendra tous les produits d'exemple, paramètres de boutique, paramètres de modules, etc.

N'utilisez pas les données sur votre site en production! Ceci remplacera tous les produits et paramètres de votre boutique.

1. Upload des fichiers de template

Pour uploader le template sur votre serveur:

  1. Ouvrez le package du template.
  2. Puis allez dans le dossier /theme/manual_install/.
  3. Uploadez les dossiers /img/, /modules/ et /themes/ dans le dossier racine PrestaShop.

Vous pouvez consulter le tutoriel vidéo détaillé sur comment uploader des fichiers sur un serveur à l'aide d'un gestionnaire FTP et comment uploader les fichiers sur un serveur avec cPanel (WebHost Manager, WHM).

2. Installation de données d'exemple

Les données d'exemple sont fournies sous la forme d'un fichier .SQL. Il se trouve dans le dossier theme/manual_install/ du package du template est il est nommé dump.sql.

Le fichier dump.sql peut être installé à l'aide de l'outil phpMyAdmin ou d'autres outils de gestion de base de données dans le panneau de contrôle de votre hébergeur. Le tutoriel détaillé sur comment installer des fichiers SQL peut vous être utile.

Avant de commencer, veuillez faites une sauvegarde de votre base de données. Vous pouvez consulter le tutoriel vidéo détaillé sur comment sauvegarder une base de données.

Assurez-vous que votre version de l'engine PrestaShop est conforme aux besoins du template. Autrement, la base de données sera endommagée.

Après avoir installé le fichier dump.sql, vous devrez activer le template.

Depuis le panneau d'administration de PrestaShop, allez dans Preferences > Themes, choisissez le template dans la section Themes et cliquez sur Save.

3. Ajout de langues à une boutique en ligne.

Une fois le template multilingue installé, vous pourrez ajouter toutes les langues qu'il propose (anglais, français, espagnol, allemand, russe) à la plate-forme de boutique en ligne. Ceci peut être fait de la manière suivante:

Allez dans 'Localization > Translations' dans le panneau d'administration, trouvez 'ADD/UPDATE A LANGUAGE' et ajoutez des langues en les choisissant dans le menu déroulant (1) et en cliquant sur le bouton 'Add or update a language' (2).

Si vous ne voulez pas de certaines langues, vous pouvez les supprimer (2) ou les désactiver (1) dans le menu 'Localization > Languages'.

4. Installation d'un template dans une boutique existante

Vous pouvez utiliser cette méthode d'installation si vous avez un site PrestaShop en production et voulez juste installer un nouveau template.

1. Upload des fichiers du template et activation du thème

  1. Ouvrez le package du template.
  2. Puis allez dans le dossier /theme/manual_install/.
  3. Uploadez les dossiers /modules/ et /themes/ dans le dossier racine PrestaShop.

    Vous pouvez consulter les tutoriels vidéos détaillés sur comment uploader des fichier sur un serveur à l'aide d'un gestionnaire FTP manager et comment upload des fichiers sur un serveur avec cPanel (WebHost Manager, WHM).

  4. Dans votre panneau d'administration PrestaShop, allez dans Preferences > Themes, sélectionnez le thème que vous avez installé (vous pouvez également utiliser le thème activé, ça n'entraînera aucun problème pour le thème activé). Cliquez sur le bouton Advanced settings.

  5. Sur l'écran qui apparait, sélectionnez le répertoire portant le nom de votre thème et cliquez sur le bouton Save.

2. Installation de modules

Chaque module est fourni avec des paramètres de module spécifiques. Pour que ces modules apparaissent comme dans la démo, vous devrez les configurer. Tous les modules sont configurables depsuis le panneau d'administration de PrestaShop admin panel.

1. Dans votre panneau d'administration PrestaShop, sélectionnez Modules > Modules dans le menu principal.

Sur la page Modules, vous pouvez voir la liste des modules PrestaShop disponibles. Vous devrez peut-être en installer certains.

Pour installer un module, trouvez-le dans la liste et cliquez sur le bouton 'Install'.

À l'aide des liens du module, vous pouvez les activer/désactiver (liens Enable|Disable), réinitialiser leurs paramètres à leur valeur par défaut (lien Reset), supprimer un module (lien Delete) ou le configurer (lien Configure).

2. Après avoir activé les modules nécessaires, vous devez leur assigner des positions correctes.

Dans le panneau d'administration PrestaShop, sélectionnez Modules > Positions dans le menu déroulant. Pour placer un module à la bonne position, glissez-le à la bonne position de la liste des modules.

3. Configuration des images

Dans PrestaShop, vous pouvez utiliser des dimensions d'images prédéfinies pour des images de produits, de catégories, de fabricants, etc.

Par exemple, les images de dimensions 'home' sont généralement utilisées pour les images de produits sur les listes par catégorie et celles de dimensions 'large' pour les images principales de produit sur la page d'informations du produit.

Pour configurer les dimensions des images de produit, ouvrez votre panneau d'administration PrestaShop, allez dans Preferences > Images et réglez les dimensions d'image comme décrit sur la Page de Configuration sur la démo live du template.

Tutoriels de base

1. Comment changer le nom de la boutique

Pour changer l'intitulé de votre boutique, ouvrez le panneau d'administration de PrestaShop et allez dans Preferences > Store Contacts, puis dans la section Contact details.

Dans le champ Shop name, entrez le nom de votre boutique et cliquez sur le bouton Save pour enregistrer vos changements.

2. Comment modifier un slider

Pour modifier les images d'un slider, trouvez le module Image slider for your homepage sur la page Modules de votre boutique PrestaShop. Il se trouve dans la section de modules Front Office Features. Cliquez sur le bouton Configure pour modifier le module.

Sur la page des paramètres de modules vous pouvez modifier les paramètres du slider (tels que la largeur, la vitesse, etc.) et modifier des slides (images de slides, URL, descriptions, etc.).

Modules et extensions personnalisés

Smart Blog

Smart blog - un ensemble de modules permettant de créer un blog dans votre boutique. Il comprend les modules suivants: Smart Blog, RSS Smart Blog Feed, Smart Blog Archive, Smart Blog Categories, Smart Blog Home Latest, Smart Blog Latest Comments, Smart Blog AddThis Module, Smart Blog Popular Posts, Smart Blog Recent Posts, Smart Blog Related Post, Smart Blog Search, Smart Blog Tags. Il a aussi ses hooks pour faire apparaître les modules de blog sur le site, ils peuvent également apparaître dans les hooks standards (colonnes uniquement).

Smart Blog

C'est le module principal contenant plusieurs paramètres pour le fonctionnement du blog. Après son installation, l'onglet du menu Blog apparaîtra dans le menu principal de votre panneau d'administration où vous pouvez ajouter/supprimer/modifier les articles/catégories ou choisir le type d'images et leurs dimensions utilisées dans les articles (les types et les dimensions des images d'articles sont différents des paramètres d'images de produit de votre boutique). Pour ajouter les paramètres front-end du module, allez dans Smart Blog (Modules->Front Office Features->Smart Blog) et ajoutez tous les paramètres.

Description des paramètres:

  1. Meta Title - le titre de votre blog qui apparaîtra dans toutes ses pages.
  2. Meta Keyword - les mots clés qio seront utilisés par les moteurs de recherche pour un meilleur référencement.
  3. Meta Description - description de votre blog utilisée par les moteurs de recherche pour un meilleur référencement.
  4. Main Blog Url - l'URL de votre Blog.
  5. Use .html with Friendly Url - activer/désactiver l'ajout de .html aux URL des pages de blog en utilisant des URL accessibles.
  6. Number of posts per page - nombre d'articles apparaissant par page (catégorie/pages d'archive, etc.).
  7. Auto accepted comment - activer/désactiver les commentaires automatiquement acceptés. S'ils sont activés, les commentaires seront disponibles sans la modération d'un administrateur.
  8. Enable Captcha - activer/désactiver le Captcha sous le formulaire Ajouter un Commentaire sur les pages de produit.
  9. Enable Comment - activer/désactiver les commentaires utilisateurs.
  10. Show Author Name - activer/désactiver l'affichage des auteurs d'articles sur les pages de blog.
  11. Show Post Viewed - activer/désactiver l'affichage du nombre de fois que les utilisateurs ont vu l'article. Il apparaîtra sous l'article sur les pages de blog.
  12. Show Author Name Style - changer le format du nom de l'auteur.
  13. AVATAR Image: - ajouter l'avatar standard pour tous les utilisateurs.
  14. Show No Image - activer/désactiver l'image. Cette option est active quand il n'y a pas d'image d'articles, de catégories, etc.
  15. Show Category - activer/désactiver l'image de catégorie et la description sur les pages des catégories du blog.
  16. Blog Page Column Setting - paramétrage de l'affichage des modules relatifs au Blog. Des paramètres supplémentaires sont souvent requis, il est donc recommandé d'installer des modules supplémentaires de la même manières que les modules PrestaShop.
  17. Custom CSS - vous pouvez ajouter des styles supplémentaires dans ce champ. Faites attention en utilisant des classes déjà utilisées dans le thème car ça peut créer des conflits.
  18. Delete Old Thumblr - utilisé pour régénérer les images de blog dans le cas où les dimensions des images ont été modifiées ou un nouveau type d'image a été ajouté. Pour régénérer, sélectionnez Yes et cliquez sur Regenerate.
C'EST SEULEMENT APRÈS CECI que d'autres modules relatifs au blog peuvent être installés. Le processus de suppression est à l'opposé VOUS DEVEZ D'ABORD SUPPRIMER TOUS LES MODULES LIÉS et, seulement après, supprimer le module Smart Blog.
RSS Smart Blog Feed

Le module générant le flux d'infos pour votre Blog.

Il présente plusieurs paramètres dans le panneau d'administration:

  1. Update Period - permet d'afficher les articles d'une période de temps donnée (heure/jour/semaine/mois/année).
  2. Update Frequency - permet de définir la fréquence des mises à jour (1 semaine, 2 heures, etc.).

Il y a en haut un chemin de fichier où les news seront générées. Il est composé de l'URL du site + /modules/smartblogfeed/rss.php.

Smart Blog Archive

Ce bloc s'affiche dans la colonne gauche/droite et sert à trier les articles par période de temps (mois) .

Il n'a aucun paramètre supplémentaire.

Smart Blog Categories

Ce bloc s'affiche dans la colonne gauche/droite et sert à trier les articles par catégorie.

Il n'a aucun paramètre supplémentaire.

Smart Blog Home Latest Posts

Ce bloc ne s'affiche que dans le hook homepagecontent et sert à afficher les derniers articles sur la page d'accueil.

Paramètres:

  1. Number of posts to display in Latest News - nombre qui s'afficheront sur la page d'accueil.
Smart Blog Latest Comments

Ce bloc s'affiche dans la colonne gauche/droite et sert à afficher les derniers commentaires publiés sur les articles de Blog.

Paramètres:

  1. Number of Comments to Show - nombre de commentaires affichés dans le bloc.
Smart Blog AddThis Module

Ce bloc s'affiche sur la page de l'article et permet de partager cet article sur les réseaux sociaux.

Il n'a pas de paramètres supplémentaires.

Smart Blog Popular Posts

Ce bloc s'affiche dans la colonne gauche/droite et sert à afficher les articles les plus populaires.

Paramètres:

  1. Number of popular Posts to Show - nombre d'articles populaires affichés dans le bloc.
Smart Blog Recent Posts

Ce bloc s'affiche dans la colonne gauche/droite et sert à afficher les derniers articles.

Paramètres:

  1. Show Number Of Recent Posts - nombre d'articles récents affichés dans le bloc.
Smart Blog Related Posts

Ce blog s'affiche dans la page de l'article et sert à afficher les articles liés.

Paramètres:

  1. Show Number Of Related Posts - nombre d'articles liés affichés dans le bloc.
Smart Blog Search

Ce bloc s'affiche dans la colonne gauche/droite et sert à rechercher dans le blog.

Il n'a pas de paramètres supplémentaires.

Smart Blog Tag

Ce bloc s'affiche dans la colonne gauche/droite et sert à afficher les tags utilisés sur le blog.

Paramètres:

  1. Number of Tag to Show - nombre de tags affichés dans le bloc.

Module Media Parallax

Qu'est-ce que l'effet parallax?

Le défilement Parallax est une technique spéciale de défilement en infographie, où les images d'arrière-plen se déplacent plus lentement par rapport à la caméra que les images au premier plan, créant ainsi une illusion de profondeur dans un jeu vidéo en 2D et augmentant l'immersion. Les web designers ont commencé à incorporer le défilement parallax en 2011, en HTML5 et en CSS3. Les sites web avec arrière-plan en parallax deviennent une stratégie de plus en plus populaire, car les avocats le présentent comme un moyen simple d'accéder à la fluidité sur le Web. De plus, les partisans utilisent des arrière-plans en parallax comme un outil pour mieux engager les utilisateurs et améliorent l'expérience générales qu'offre le site. Le défilement Parallax permet la création d'une expérience utilisateur dynamique et intéractive pouvant potentiellement améliorer l'attrait visuel.

Ce module permet d'ajouter des effets parallax à votre site web sans aucun effort ni compétence supplémentaire. Il est possible d'ajouter à la fois des images et une vidéo en arrière-plan.

Paramètres: (uniquement pour Google Chrome )

  1. Enable smooth scroll for Google Chrome browser - active un effet supplémentaire.
  2. Smooth scroll time - durée de l'effet de défilement.
  3. Smooth scroll distance - distance de l'effet de défilement.

Ajout d'un Média module parallax

  1. Vous devez ajouter l'id ou la classe dans le contenu de votre page HTML. Vous pouvez également utiliser une balise unique, cet élément ne doit pas être répété..
  2. Créez l'image de taille requise (il est préférable de créer une image de taille supérieure à celle du bloc où l'image sera ajoutée). Dans le cas où vous voulez ajouter une vidéo, vous devez la préparer en 3 formats: .mp4,.webm et .ogv, et vous devez aussi ajouter une image d'aperçu de la même taille (.png, .jpg et .gif)

  3. Cliquez sur le bouton Add container dans Add new block.

  4. Sur l'écran qui apparaît, ajoutez tous les paramètres requis.

    Paramètres:

    1. Sélecteur - l'élément sur lequel l'effet sera ajouté (id ou balise unique).
    2. Parallax type - le type d'effet qui sera utilisé (image ou vidéo).
    3. Media files - uploadez tous les fichiers requis.
    4. Cliquez sur le bouton Save.

Personnalisation du module média parallax

  1. Dans la liste des blocs Parallax, trouvez le bloc requis et changez les paramètres.

Suppression du module média parallax

  1. Dans la liste des blocs Parallax, trouvez les blocs requis et cliquez sur le bouton Delete.

Module de Vidéo en Arrière-Plan

Ce module permet d'ajouter l'arrière-plan vidéo à n'importe quel bloc de votre page HTML (il doit y avoir l'id ou la balise unique).

Pour ajouter la vidéo d'arrière-plan, vous pouvez suivre les étapes ci-dessous:

  1. Uploadez les fichiers vidéos et les images d'aperçu dans le dossier modules/tmvideobackground/video (ou n'importe quel autre dossier du serveur) par FTP. Les vidéos doivent être uploadées en 3 formats: .mp4, .ogv, .webm, tous les fichiers ayant les mêmes noms (ocean.mp4, ocean.ogv, ocean.webm). Les images d'aperçu doivent elles-aussi être en 3 formats: .jpg, .gif, .png et avoir les mêmes noms (ocean.jpg, ocean.gif, ocean.png).
  2. Dans votre panneau d'administration, ouvrez Video background module (Modules->Front Office Features->Video background module) et réglez les paramètres requis.

    Paramètres:

    1. Video selector - l'élément de la page HTML auquel la vidéo d'arrière-plan sera ajoutée (id ou balise unique (body, header, footer, etc.).
    2. Video path - le chemin vers vos fichiers vidéo (le chemin doit avoir / à la fin), les fichiers peuvent se trouver dans n'importe quel dossier sur le serveur, mais nous recommandons de les uploader dans le dossier modules/tmvideobackground/video/.
    3. Video files name - les noms de vos fichiers vidéo. Seul le nom doit être ajouté (il n'y a aucun besoin d'écrire le format. Par exemple,"ocean").
    4. Cliquez sur le bouton Save.
Pour afficher la vidéo, aucun autre arrière-plan ne doit être ajouté à cet élément.

Contenu TM HTML

Ce module permet d'afficher n'importe quel contenu (image, HTML, liens, etc.) aux positions suivantes: displayNav, home, top, topColumn, left and right (displayLeftColumn/displayRightColumn), footer. Il sert à étendre les options de design et à ajouter des blocs de publicité/information à votre boutique. C'est un module multilingue et il peut aussi fonctionner dans les sites à plusieurs boutiques.

Pour ajouter le contenu, vous devez suivre les étapes ci-dessous:

  1. Allez dans Modules->Front Office Features->TM HTML Content (si ce n'est pas installé, installez-le).

    1. Cliquez sur le bouton Add new.
    2. La liste des langues disponibles. Vous pouvez voir les éléments disponibles pour toutes les langues en naviguant entre les onglets des langues.
    3. La liste d'éléments ajoutés pour une position et une langue précises.

  2. Ajout de l'élément. Cliquez sur le bouton Add item et remplissez les champs de paramètres (aucun des champs n'est obligatoire).

    1. Language - sélectionnez le langue requise dans la liste déroulante.
    2. Title - titre de l'élément.
    3. Use title in front - afficher le titre de l'élément en avant (le titre s'affichera devant l'élément).
    4. Hook - la position où l'élément est ajouté.
    5. Image - uploader l'image depuis votre ordinateur.
    6. Image width - largeur de l'image présentée.
    7. Image height - longueur de l'image présentée.
    8. URL - ajouter le lien (le bloc tout entier sera enveloppé par le lien).
    9. Target blank - ouvrir le lien dans un nouvel onglet.
    10. HTML - code HTML (sauf le lien si vous l'avez déjà ajouté dans le champ URL) ou texte.
    11. Save - cliquez pour enregistrer l'élément.
    Tous les éléments d'une langue ajoutée à la même position seront affichés sous forme de liste en back-end et en front-end.

  3. Modifier et supprimer l'élément

    Quand vous ajoutez l'élément, il s'affiche dans la liste correspondante.

    1. La langue dans laquelle les éléments ont été ajoutés.
    2. La position à laquelle les éléments ont été ajoutés.
    3. Le bouton de Modification avec la liste déroulante.
    4. Le bouton de Suppression.

     

    Modification

    1. Cliquez sur le bouton Edit pour modifier l'élément.

      Dans l'écran apparu, modifiez tous les champs requis.

      1. Enable - afficher l'élément en front-end.
      2. Image title - nom de l'élément.
      3. Use title in front - afficher le nom en front-end.
      4. Hook to which the image should be attached - la position de l'élément
      5. Load your image - l'image de l'élément.
      6. Image width - largeur de l'image en front-end.
      7. Image height - la hauteur de l'image en front-end.
      8. Target link - assigner un lien à l'élément.
      9. Open link in a new tab/page - Ouvrir le lien dans un nouvel onglet.
      10. Optional HTML code - modifier le code HTML.
      11. Save - enregistrer les changements.

  4. Changement de l'ordre des éléments de la liste

    Pour changer l'ordre des éléments, vous devez glisser l'élément à la position que vous souhaitez.

TM Categories Badges

This module allows to add badges to the sidebar categories of the default Prestashop Categories module ("Blockcategories"). The badges can be added to the category of any level and in any language. The text field of the label is mandatory and should be filled in all languages. If this field is left blank in any language, then it will automatically take the text of the default language.

The module is installed and deleted like any other Prestashop module.

Module Configuration

After installation you can change its settings on the module configuration page.

You can add badges to any category after selecting it from the drop down list. Attention! You can add only one badge for each category. Then, you can add text that will show up in the badge and select the style from the drop-down list (the styles may differ in different templates).

You can see the list of existing badges below. Here you can edit or delete badges with the help of corresponding buttons.

The module is multilanguage and supports the multi-stores. You can add badges for group of stores or all stores at once. They will show up in the list as separate elements.

TM Google Map

This module allows to add Google map with your stores location and short description about them on your front page. The map will show up the addresses of all stores related to your online shop. You can edit your stores details under Preferences->Store Contacts tab.

Installation and Deleting

The module is installed and deleted like any other Prestashop module.

Module configuration

After installation you can change its settings on the module configuration page.

You can configure the style and elements of your map.

Select the map style from the first list (all styles are taken from Snazzymaps). You can also add the style from this website. Find instructions on adding them at the end of this instruction.

Select the view type from the second list (satellite view or street view).

You can also select the zoom ratio for your map from 1 to 17.

Zoom option if scrolling on the map area.

You can disable the map control and ability to view the streets.

The module supports the multi-stores. You can change the map styles for each store.

Adding custom styles

You can add any custom style from Snazzymaps to your module. To do this, you should select the style from the website, copy JavaScript Style Array to .js file and save it in the module folder /js/styles/(for the file name use the style name replacing the spaces with underscores and removing all special characters. For example: for Subtle Grayscale style - subtle_grayscale.js file, for Apple Maps-esque style - apple_maps-esque.js file). Then, navigate to your module configuration page and select the required style from the styles drop-down list.

TM Manufacturers Block

This module shows up manufacturers available in your store. The list of manufacturers may consist of the image and title. The list can also show up as a carousel.

Installing and Deleting

The module is installed and deleted like any other Prestashop module.

Module configuration

After installation you can change its settings on the module configuration page.

  1. Display name - show the manufacturer's name.
  2. Display image - show the image of the manufacturer.
  3. Number of elements to display - select the number of elements displayed in the module.
  4. Image Type - select the image type (you can select the image from the images related to the manufacturer).
  5. Use carousel - show up the list as a carousel.
  6. * Visible items - number of visible manufacturers.
  7. * Items scroll - number of manufacturers shown in one carousel scroll.
  8. * Slide Width - the width of the element in the carousel.
  9. * Slide Margin - the margin of the element in the carousel.
  10. * Auto scroll - automatic scrolling.
  11. * Carousel speed - speed of the carousel.
  12. * Pause - pause between scrolls(if "Auto scroll" is enabled).
  13. * Random - carousel start from the random element.
  14. * Carousel loop - carousel returns back to the beginning when all elements have been scrolled.
  15. * Hide control on end - hide next/previous buttons.
  16. * Pager - show pagination.
  17. * Control - show next/previous buttons.
  18. * Auto control - show play/pause buttons.
  19. * Auto hover - pausing when hovering over carousel elements.

* - settings are available only if carousel is enabled.

The module supports multi-stores.

TM Products Slider

The module shows up store products in the slider.

Installation and deleting

The module is installed and deleted like any other Prestashop module.

After installation you can change its settings on the module configuration page.

You can set the order of products in your slider using drag-and-drop method. You can also enable/disable product display in the slider.

You can set the width of slider, sliding effect, slider speed, pause, auto scroll, pause when hovering over carousel elements, slider control, next/previous buttons, play/pause buttons.

After module installation you will see the new tab in your product configuration page (Catalog-> Product-> select any product - TM Products Slider).

Here you can set whether the product will show up in the slider or not. After making the changes, click Save/Save and Stay button to save them.

The module supports multi-stores. You can change settings for each store, group of stores or all stores at a time.

TM Product Videos

This module allows to add unlimited number of videos to any product in any language and store.

Installation and deleting

The module is installed and deleted like any other Prestashop module. It has no configuration page and supports multi-stores.

After module installation you will see the new tab in your product configuration page (Catalog-> Product-> select any product - TM Product Videos).

To add a video to any product, you should fill in the required fields (the link field is mandatory). The module is multilanguage, so you will have to fill in the fields for all languages this video is related to. If the Link field is not filled for any language, then the video will not be added for this language. The module is supposed to show up videos from Youtube and Vimeo, so you need to add links only from these resources. After making the changes, click Save/Save and Stay button to save them.

Examples of links

To display video properly, the links should have the following format:

  • Youtube - https://www.youtube.com/v/K6VcTyaym90

  • Vimeo - http://player.vimeo.com/video/66140585

If the product has video related to this language, you will see the list of videos below.

To save, delete or edit the list, there is no need to click Save/Save and Stay button. Use the buttons located in the video blocks. To change the order of videos, use drag-and-drop method to change their position. To enable/disable the video, use the button next to video. To edit the videos, fill in the fields with required data and click Update button.

To remove the video from the list, click Remove video button.

While switching languages, only the list of videos related to the language will show up.

To view the video in zoomed format, click Zoom button under the video you want.

Politique de Cookies

Les cookies sont de petits fichiers texte que les sites mettent dans les ordinateurs des visiteurs. Ils sont généralement utilisés pour identifier un utilisateur et lui proposer une meilleure expérience. Les cookies sont un élément crucial pour plusieurs fonctions essentielles des sites web.

Par exemple, ce sont généralement les cookies qui permettent aux boutiques en ligne de se souvenir des articles que vous avez dans votre panier d'achat. Ils permettent aussi de vous garder connecté à un site, ou de fournir des statistiques et des informations riches aux propriétaires de sites web.

Si vous possédez un site web, il utilise à coup sûr des cookies. Leur fonction la plus répandue est le suivi du nombre et du comportement des visiteurs à l'aide d'outils comme Google Analytics. Ils peuvent aussi servir à afficher des publicités pertinentes aux visiteurs ou - si vous vendez en ligne - à renforcer des secteurs clés de votre système d'achat en ligne.

26 Mai 2012 - L'ICO a apporté un changement à la loi, le "consentement implicite" est maintenant autorisé tant qu'on indique clairement où se trouve sa politique de vie privée/cookies.

Pour vous faciliter la tâche, nous avons ajouté un module Cookieslaw à notre thème.

Sa configuration est très simple.

  1. Vous le trouverez dans le dossier manual_install qui se trouve dans le package du template (si vous avez fait l'installation manuelle du template avec des données d'exemple, le module sera automatiquement ajouté).

  2. Trouvez-le ensuite dans la liste de modules de votre panneau d'administration (le module Cookieslaw est dans les modules "Front Office Features") et cliquez sur le bouton Install.

  3. Après l'installation (ignorez cette étape si vous avez été directement dirigé vers la page de configuration du module) cliquez sur le bouton Configure.

  4. Ajoutez tous les paramètres requis et cliquez sur le bouton Save.

    Paramètres:
    • - Afficher sur le haut de la page (par défaut). Si vous décochez cette option, le module apparaîtra au bas de la page.
    • - Lien vers la Politique de Cookies.
    • - Lien vers la page où l'utilisateur est redirigé dans le cas où il n'accepte pas la Politique de Cookies.

Nous pouvons également recommander les modules suivants:

  • Module EU Cookie Law
  • Module Alert for EU Cookie Law

Chargement de données depuis les serveurs Google

Certains pays ont des difficultés pour se connecter aux serveurs Google. Ceci peut entraîner des problèmes de chargement de Google Fonts et de bibliothèques JavaScript depuis Google CDN. Pour résoudre ces problèmes, veuillez suivre les instructions suivantes:

Impossible de charger des Google Fonts:

  1. Ouvrez le fichier \themes\theme****\header.tpl.

  2. Trouvez la ligne: 

    <link rel="stylesheet" href="http{if Tools::usingSecureMode()}s{/if}://fonts.googleapis.com/css?family=[---your_web_font_name_here---]" type="text/css" media="all" />

  3. Remplacez-la par la suivante: 

    <link rel="stylesheet" href="http://fonts.useso.com/css?family=[---le_nom_de_votre_web_font---]" type="text/css" media="all" />
