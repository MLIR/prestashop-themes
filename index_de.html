Einführung

Was ist eigentlich die PrestaShop Plattform?

PrestaShop ist eine kostenlose Plattform für den Internethandel mit dem Open Source. Sie unterstützt Zahlungssysteme wie Google checkout, paypal oder payments pro per APIs.

PrestaShop ist unter den Bedingungen der Open Source Software (Open Software Licence) verfügbar und wurde im August 2007 offiziell für kleine und mittlere Unternehmen gestartet. Diese Plattform wurde auf der Basis von Smarty Engine entwickelt, die Tausende Online-Shops weltweit verwenden.

Was ist eigentlich eine PrestaShop Vorlage?

PrestaShop Vorlage ist das Thema für die Internethandel-Plattform PrestaShop. Sie bietet breite Möglichkeiten zur Bearbeitung und ermöglicht eine Online-Shop zu erstellen, der völlig Ihren Anforderungen entsprechen wird.

Dateistruktur

Die gekaufte Vorlage besteht aus einigen Ordnern. Lassen Sie uns den Inhalt jedes Ordners überprüfen:

  • screenshots - enthält Bildschirmfotos des Themas. Für die Arbeit mit der Vorlage braucht man sie nicht.
  • sources.zip - enthält die Quelldateien des Themas.
    • sources
      • psd - sind Dateien der Vorlage im Format Adobe Photoshop (.psd)
  • theme - enthält die Dateien des Themas PrestaShop
    • manual_install - Dateien für manuelle Installation
      • img - Bilder für manuelle Installation
      • modules - Module für manuelle Installation
      • themes - Dateien des Themas für manuelle Installation
      • dump.sql - Datei mit den Demodaten für manuelle Installation
    • sample_data - enthält die Dateien für die Installation mit den Demodaten mit Hilfe vom Modul sampledatainstall
    • themeinstallator - enthält das Archiv für automatische Installation des Themas
      • theme####.zip - Archiv des Themas. Es enthält alle Dateien des Themas. Es wird mit Hilfe von der Standard-Installationsmethode in PrestaShop installiert
  • fonts_info.txt - Datei mit Links zum Herunterladen von Schriften, die in der Vorlage verwendet werden.
  • info.txt - Anleitung zum Entpacken von Quelldateien

Arbeitsbeginn

Vorbereitung

Bebor Sie anfangen die PrestaShop Webseite zu installieren, müssen Sie alles völlig vorbereiten. Führen Sie, bitte, die folgenden Schritte aus:

Software

Um mit der Vorlage arbeiten zu können, müssen Sie alle notwendigen Anwendungen herunterladen. Die Liste von notwendigen Softwares befindet sich auf der Vorschauseite der Vorlage. Die Anforderungen an verschiedenen Vorlagen können sich unterscheiden, deswegen nennen wir nur die wichtigsten:

  1. Vor allem brauchen Sie die richtigen Anwendungen, um passwortgeschütztes Archiv sources_#########.zip zu entpacken.Sie können WinZip 9+ (Windows) und Stuffit Expander 10+ (Mac) nutzen.
  2. Sie können auch das Programm Adobe Photoshop benötigen. Es wird benutzt, um die Quelldateien .PSD zu bearbeiten oder das grafische Design und Bilder der Vorlage zu ändern.
  3. Um den Quellcode der Vorlagendateien zu bearbeiten, müssen Sie den Code-Editor wie Adobe Dreamweaver, Notepad++, Sublime Text usw. nutzen.
  4. Um die Dateien auf den Hosting-Server hochzuladen, brauchen Sie den FTP-Kunden wie Total Commander, FileZilla, CuteFTP usw.

Anforderungen an Server

Überzeugen Sie sich, bitte, dass Ihr Hosting-Server den PrestaShop Anforderungen entspricht:

  1. System: Unix, Linux oderи Windows. Empfehlenswert ist Unix.
  2. Webserver: Apache Web server 1.3 oder höher, Microsoft's IIS Web server 6.0 oder höher, und nginx 1.0 oder höher.
  3. PHP 5.2* oder höher.
  4. MySQL 5.0 oder höher.
  5. Mindestens 32 MB Arbeitsspeicher auf dem Server (empfehlenswert ist 64 Mb, je mehr, desto besser).
PrestaShop funktioniert mit PHP 5.1 oder höher, weil die Versionen vor 5.2 Fehler enthalten. Aus diesem Grund funktionieren einige Funktionen nicht korrekt (zum Beispiel, falsches Datum der Zeitzonen).

Verwendung des lokalen Servers

PrestaShop kann auf dem Personalcomputer mit Hilfe von dem lokalen Server installiert werden. Um den lokalen Hosting-Server zu erstellen, sollen Sie die Softwares wie WAMP, AppServ, MAMP usw. nutzen. Jede von ihnen wird wie normales Programm installiert, das PrestaShop unterstützt.

Sie können die folgenden Tutorials nutzen, um den lokalen Server einzustellen:

  • Wie installiert man die Web-Entwicklungsumgebung AppServ
  • Wie installiert man die Web-Entwicklungsumgebung WAMP
  • Wie installiert man die Web-Entwicklungsumgebung XAMP

Herunterladen

Um PrestaShop zu installieren, laden Sie die PrestaShop Engine von der offiziellen Website herunter www.prestashop.com/en/downloads

Geben Sie den Namen und die E-Mail-Adresse ins Downloadfeld ein und drücken Sie auf den Button 'Herunterladen'.

Überzeugen Sie sich, bitte, dass die Version PrestaShop den Vorlagenanforderungen entspricht. Alle Versionen PrestaShop sind unter dem Link SourceForgeverfügbar.

Entpacken von Dateien

Nachdem Sie die PrestaShop Engine heruntergeladen haben, sollen Sie die Dateien und Ordner aus dem Archiv entpacken.

Dies können Sie mit Hilfe von jeglichem Archivierungsprogramm, zum Beispiel, WinZip(PC), StuffitExpander(MAC), usw. machen.

Nutzen Sie die folgenden Tutorials, um die Archive mit Hilfe von WinZip(PC) und StuffitExpander(MAC) zu entpacken.

Achtung: wenn es im Control Panel desHosting die Option zur arbeit mit Dateien 'Entpacken'gibt, können Sie sie nutzen, um die Dateien auf dem Server nach dem Hochladen von .zip Archiven zu entpacken.

Hochladen von Dateien der PrestaShop Engine

Wenn Sie das Archiv lokal (auf Ihrem Computer) entpackt haben, sollen Sie die Dateien aufs Hosting hochladen.

Dafür können Sie den Dateimanager im Control Panel des Hostings oder eine von den FTP Anwendungen wie: FileZilla, TotalCommander, CureFTP usw. nutzen.

Sie können auch die ausführliche Video-Anleitung anschauen Wie lädt man die Dateien auf den Server per FTP-Kunde hoch und Wie lädt man die Dateien per cPanel (WebHost Manager, WHM) hoch.

Installation der PrestaShop Engine

Nachdem Sie alle Dateien hochgeladen haben, können Sie anfangen die PrestaShop Engine zu installieren. Öffnen Sie den Webbrowser und geben Sie den Domainnamen oder den Pfad zum Verzeichnis PrestaShop in der Adresszeile ein, dann aktualisieren Sie die Seite. Es sollte das Begrüßungsfenster angezeigt werden. Folgen Sie den folgenden Anleitungen, um PrestaShop zu installieren.

Schritt 1: Installer (Wählen Sie die Sprache)

Wählen Sie die Installationssprache

Achtung: die Installationssprache wirkt auf Spachen des Online-Shops nicht. Standardmäßig wird in PrestaShop Englisch installiert. Sie können zusätzliche Lokalisierungspakete installieren, um die Sprache des Online-Shops nach der Installation PrestaShop zu ändern

Nachdem Sie die Sprache ausgewählt haben, drücken Sie auf den Button 'Weiter'

Schritt 2: Installer (Lizenzvereinbarung)

Machen Sie sich mit der Lizenzvereinbarung bekannt.

Dann drücken Sie auf den Button 'Weiter'.

Schritt 3: Systemkompatibilität

Überprüfen Sie PHP Einstellungen und Verzeichnisberechtigungen. Falls es Fehler gibt, setzen Sie sich mit dem Hosting-Anbiert in Verbindung, damit er Ihnen helfen kann.

Wenn es keine Fehler gibt, drücken Sie auf den Button 'Weiter'.

Schritt 4: Einstellung des Online-Shops

Jetzt müssen die Einstellungen des Online-Shops wie Name, Tätigkeit, Land, Zeitzone usw. festgelegt werden.

Alle diesen Einstellungen können Sie später im Control Panel PrestaShop ändern.

Schritt 5: Systemeinstellungen

Wenn Sie die Datenbank MySQL schon haben, geben Sie die Details in entsprechende Felder ein.

Ansonsten können Sie die Details der Datenbank im Control Panel des Hosting finden oder den Kontakt mit den Hosting-Anbieter aufnehmen.

Danach drücken Sie auf den Button 'Weiter'.

Schritt 6: Installation ist abgeschloßen

Der letzte Installationsschritt.

Aus Sicherheitsgründen muss der Ordner /install/ auf dem Server entfernt werden.
Ab der Version 1.5.4 benennt PrestaShop den Administrationsordner automatisch um: der Name sieht wie "admin" mit 4 Zufallszahlen, zum Beispiel, "admin8814" aus. Die Umbenennung wird durchgeführt, sobald Sie zum ersten Mal den Ordner "admin" im Browser öffnen: PrestaShop benennt ihn um und öffnet dann schon umbenannten Ordner.

Loggen Sie sich ins Control PanelВ PrestaShop ein, indem Sie umbenanntes Verzeichnis "admin" nutzen, und fangen Sie an, den Katalog mit Ihren Daten zu füllen.

Falls Sie auf Schwierigkeiten stoßen, nutzen Sie die offizielle Anleitung zur PrestaShop Installation.

Installation der Vorlage

1. Import des Themas

Achtung: diese Methode wird nicht Demodaten installieren.

1. Vorbereitung der VorlageПодготовка шаблона.

Bevor Sie anfangen, überzeugen Sie sich, dass Sie die Vorlagendateien heruntergeladen und entpackt haben. Sie können ausführliche Tutorials nutzen, um die Dateien aus dem Archiv der Vorlage mit Hilfe von WinZip und StuffitExpander zu entpacken.

Nachdem Sie die Dateien entpackt haben, können Sie die Vorlage hochladen.

2. Installation.

Loggen Sie sich ins Control Panel PrestaShop ein und gehen Sie in Preferences -> Themes (Einstellungen -> Themata). In der oberen Ecke drücken Sie den Button Add new theme (Neues Thema hinzufügen).

Klicken Sie auf den Button Add file (Datei hinzufügen) im Bereich Import from your computer (Import von Ihrem Computer). Finden Sie die entpackte Vorlage und öffnen Sie den Ordner themeinstallator.

Wählen Sie die .zip Datei des Themas und drücken Sie Save (Speichern).

Der Hochladen-Prozess wird gestartet. Dann werden Sie die Mitteilung sehen, dass das Thema erfolgreich hochgeladen wurden. Das Icon des neuen Themas wird in der Liste von installierten Themeten dargestellt.

Um das Thema zu aktivieren, positionieren Sie den Cursor über das Icon des Themas und drücken Sie Use this theme (Dieses Thema verwenden).

Sie werden das Fenster mit der Information über verwendete im Thema Module und Anzahl von Modulen, die aktiviert oder deaktiviert werden, sehen. Um fortzufahren, klicken Sie Save (Speichern).

Danach wird das DropDownFenster mit der Mitteilung angezeigt, dass das Thema erfolgreich aktiviert wurde: das Thema wurde erfolgreich installiert und Bilder zu festgelegten im Thema Berechtigungen angepasst. Um die Installation anzuschließen, drücken Sie auf den Button Finish (Abschließen).

2. Automatische Installation mit den Demodaten (Modul Sample data installer)

Nachdem Sie Demodaten installiert haben, wird Ihr Online-Shop PrestaShop wie auf der Demoversion aussehen. Er wird alle Produkte, Kategorien und Einstellungen für Module und den Online-Shop enthalten.

Nutzen Sie, bitte, die Demodaten für aktuelle Webseite nicht, weil alle Produkte und Einstellungen überschrieben werden.

Um die Demodaten mit Hilfe vom Modul модуль Sample data installer zu installieren, folgen Sie der Anleitung aus dem Schritt 1 (Import des Themas) dieser Dokumentation. Dann wird im Control Panel des Online-Shops das Modul Sample data installer angezeigt.

Mit Hilfe von diesem Modul kann man einfach die Demodaten installieren. Machen Sie das Folgende:

  1. Im Control Panel drücken Sie "Install Demo Store (Demo-Shop installieren)".

  2. Dann sollen Sie die Dateien von Demodaten aus dem Ordner 'sample_data' markieren und hochladen. Dies können Sie anhand des Buttons "Browse local files (Dateien finden)" machen oder die Dateien in das Browserfenster verschieben.

  3. Warten Sie bis das Hochladen von Dateien abgeschlossen wird.

  4. Drücken Sie den Button "Continue Install (Installation fortfahren)".

  5. Danach wird das Modul Sample data installer automatisch den Online-Shop und andere Module einstellen. Es kann einige Minuten dauern.

  6. Nachdem Sie die Demodaten installiert haben, müssen Sie noch eine Aktion durchführen. Regenerieren Sie die Bilder. Drücken Sie den Button "Go To Regenerate Page (Zur Regenerationsseite gelangen)".

  7. Um die Bilder zu regenerieren, klicken Sie den Button "Regenerate thumbnails (Vorschaubild regenerieren)".

Wir gratulieren Ihnen, der Demo-Shop wurde erfolgreich installiert.

3. Manuelle Installation der Vorlage mit Demodaten

Nachdem Sie Demodaten installiert haben, wird Ihr Online-Shop von PrestaShop wie auf der Demoversion aussehen. Er wird alle Produkte, Kategorien, Einstellungen für Module und den Online-Shop enthalten.

Nutzen Sie, bitte, keine Demodaten für aktuelle Webseite! Dies wird alle Produkte und Einstellungen des Online-Shops überschrieben.

1. Hochladen von Vorlagendateien

Um die Dateien auf den Server hochzuladen:

  1. Öffnen Sie den Ordner Ihrer Vorlage.
  2. Gehen Sie in den Ordner /theme/manual_install/.
  3. Laden Sie die Ordner /img/, /modules/ und /themes/ ins Stammverzeichnis PrestaShop hoch.

Sie können auch die ausführlichen Video-Tutorials anschauen wie lädt man die Dateien auf den Server per FTP-Kunde hoch und wie lädt man die Dateien auf den Server per cPanel (WebHost Manager, WHM) hoch.

2. Installation von Demodaten

Die Demodaten befinden sich in der Datei .SQL. Sie befindet sich im Ordner theme/manual_install/ Ihrer Vorlage und heißt dump.sql.

Die Datei dump.sql kann man mit Hilfe von phpMyAdmin oder anderen Datenbank-Verwaltungswerkzeugen im Control Panel des Hostings installieren. Diese ausführliche Anleitung kann für Sie hilfreich sein wie importiert man SQL Dateien.

Bevor Sie anfangen die Dateien zu importieren, erstellen Sie die Sicherungskopie der Datenbank. Sie können sich das ausführliche Video-Tutorial darüber anschauen wie erstellt man die Sicherungskopie der Datenbank.

Überzeugen Sie sich, dass die Version der PrestaShop Engine den Anforderungen Ihrer Vorlage entspricht. Andernfalls wird die Datenbank beschädigt.

Nachdem Sie die Datei dump.sql importiert haben, sollen Sie die Vorlage aktivieren.

Im Control Panel PrestaShop gehen Sie in Preferences > Themes (Einstellungen > Themen), wählen Sie die Vorlage aus der Liste Themes (Themen) und drücken Sie Save (Speichern).

3. Hinzufügen von Sprachen zum Online-Shop.

Nachdem Sie mehrsprachige Vorlage installiert haben, können Sie zum Ihren Online-Shop jegliche Sprachen, die verfügbar sind, (Englisch, Französisch, Spanisch, Deutsch, Russisch) hinzufügen. Dies können Sie wie folgt machen:

Gehen Sie in 'Localization > Translations (Lokalisierung> Übersetzungen)' im Control Panel, finden Sie 'ADD/UPDATE A LANGUAGE (SPRACHE HINZUFÜGEN/AKTUALISIEREN)', markieren Sie die Sprachen aus dem DropDownMenü (1) und drücken Sie den Button 'Add or update a language (Sprache hinzufügen oder aktualisierenк)' (2).

Wenn Sie irgendwelche Sprachen nicht benötigen, können Sie (2) sie (1) im Menü 'Localization > Languages (Lokalisierung> Übersetzungen)' entfernen oder deaktivieren.

4. Installation der Vorlage auf den aktuellen Online-Shop

Diese Installationsmethode müssen Sie in dem Fall nutzen, wenn Sie schon eine aktuelle Webseite von PrestaShop haben und nur die Vorlage installieren möchten.

1. Hochladen von Vorlagendateien und Aktivierung des Themas

  1. Öffnen Sie den Ordner Ihrer Vorlage.
  2. Gehen Sie in den Ordner /theme/.
  3. Laden Sie die Ordner /modules/ und /themes/ ins Stammverzeichnis PrestaShop hoch.

    Sie können sich die ausführlichen Video-Tutorials darüber anschauen wie lädt man due Dateien auf den Server per FTP-Kunde hoch und wie lädt man due Dateien auf den Server per cPanel (WebHost Manager, WHM) hoch.

    Im Control Panel PrestaShop gehen Sie in Preferences > Themes (Einstellungen > Themen), wählen Sie das installierte Thema (auch können Sie das aktuelle Thema wählen). Drücken Sie auf den Button Advanced settings (Erweiterte Einstellungen).

  4. In dem DropDownFenster legen Sie das Verzeichnis mit dem neuen Thema fest und drücken Sie auf den Button Save (Speichern).

2. Installation von Modulen

Jede Vorlage enthält bestimmte Einstellungen für Module. Damit die Module wie auf der Demoversion aussehen, müssen sie eingestellt werden. Alle Module werden im Control Panel PrestaShop eingestellt.

1. Im Control Panel PrestaShop wählen Sie Modules > Modules (Module > Module) im Hauptmenü.

Auf der Modulseite befindet sich die Liste von allen verfügbaren Modulen PrestaShop. Einige von ihnen müssen installiert werden.

Um das Modul zu installieren, finden Sie es in der Liste und drücken Sie auf den Button 'Install (Installieren)'.

Indem Sie die Links des Moduls nutzen, können Sie es aktivieren/deaktivieren (Links Enable (Aktivieren)|Disable (Deaktivieren)), auf die Vorgaben zurücksetzen (Link Reset (Zurücksetzen)), das Modul (Link Delete (Entfernen)) entfernen oder (Link Configure (Einstellen)) einstellen.

2. Nachdem Sie notwendige Module aktiviert haben, sollen Sie sie in der richtigen Position platzieren.

Im Control Panel PrestaShop wählen Sie Modules > Positions (Module > Positionen) im Hauptmenü. Um das Modul in der notwendigen Position zu platzieren, ziehen Sie es einfach im Block in der Liste der Module.

3. Anpassen von Bildern

In PrestaShop kann man festgelegte Berechtigungen für Produktbilder, Kategorien, Hersteller usw. nutzen.

Normalerweise werden die Bildergrößen wie 'home' für Bilder in der Kategorienliste 'large' für Hauptproduktbild auf der Produktseite benutzt.

Um die Bildauflösungen einzustellen, öffnen Sie das Control Panel PrestaShop, gehen Sie in Preferences > Images (Einstellungen > Bilder) und installieren Sie die Bildergröße, wie es auf der Seite Configuration Page (Vorlageneinstellungen) Gemo geschrieben ist.

Haupttutorials

1. Wie ändert man den Shopnamen

Um den Shopnamen zu ändern, öffnen Sie das Control Panel PrestaShop und gehen Sie in Preferences > Store Contacts (Einstellungen > Kontaktdaten des Online-Shops), Bereich Contact details (Kontaktdaten).

Im Feld Shop name (Shopname) geben Sie den Namen für den Online-Shop ein und drücken Sie den Button Save (Speichern) um die Änderungen zu speichern.

2. Wie bearbeitet man den Slider

Um die Bilder des Sliders zu bearbeiten, finden Sie das Modul Image slider for your homepage (Diashow für die Hauptseite) auf der Seite Modules (Module) des Online-Shops von PrestaShop. Es befindet sich im Bereich Front Office Features (Funktionen front-office). Klicken Sie den Button Configure (Einstellen), um das Modul zu bearbeiten.

Auf der Seite der Moduleinstellungen kann man die Optionen für den Slider (zum Beispiel, Breite, Geschwindigkeit, usw.) ändern und das Dia (Bilder für den Slider, URL, Beschreibung usw.) zu bearbeiten.

Benutzerdefinierte Module und Erweiterungen

Smart Blog

Smart blog - ist ein Modulsatz, der ermöglicht, den Blog im Online-Shop zu erstellen. Er besteht aus den folgenden Modulen: Smart Blog, RSS Feed Smart Blog, Archiv Smart Blog , Kategorien Smart Blog , Neues auf der Hauptseite, Neueste Kommentare Smart Blog, Modul für soziale Netzwerke Smart Blog , Beliebte Beiträge Smart Blog, Neueste Beiträge Smart Blog, Ähnliche Beiträge Smart Blog, Suche Smart Blog, Tags Smart Blog. Er enthält auch eigene Blöcke, in denen Module des Blogs auf der Webseite dargestellt werden. Dazu können sie in Standardblöcken (nur Spalten) platziert werden.

Smart Blog

Es ist das Hauptmodul mit verschiedenen Einstellungen für den Blog. Nachdem Sie es installiert haben, wird im Control Panel im Hauptmenü ein neuer Tab Blog dargestellt, wo Sie Beiträge/Kategorien hinzufügen/entfernen/bearbeiten oder Arten und Bildauflösung, die in den Beiträgen verwendet werden, (Arten und Bildauflösung von Beiträgen unterscheiden sich von Einstellugen für Bilder im Online-Shop) wählen können. Um die Einstellungen zum Moduldarstellung auf der Webseite hinzuzufügen, gehen Sie in Smart Blog (Modules->Front Office Features->Smart Blog) ((Module->Funktionen front-office->Smart Blog)) und fügen Sie alle Optionen hinzu.

Parameterbeschreibung:

  1. Meta Title (Meta-Titel) - Titel des Blogs, der auf allen Seiten dargestellt wird.
  2. Meta Keyword (Meta-Schlüsselwörter) - Schlüsselwörter, die von Suchmaschinen für bessere SEO-Optimierung benutzt werden.
  3. Meta Description (Meta-Beschreibung) - Blogbeschreibung, die von Suchmaschinen für bessere SEO-Optimierung benutzt werden
  4. Main Blog Url (Haupt-URL des Blogs) - URL Ihres Blogs.
  5. Use .html with Friendly Url (.html zum freundlichen Link hinzufügen) - aktiviert/deaktiviert die Hinzufügung .html zum freundlichen Link.
  6. Number of posts per page (Anzahl von Beiträgen pro Seite) - Anzahl von Beiträgen, die auf der Seite dargestellt werden (Kategorieseiten/Archivs usw.).
  7. Auto accepted comment (Automatische Hinzufügung von Kommentaren) - aktiviert/deaktiviert automatische Hinzufügung von Kommentaren. Wenn Sie diesen Parameter aktivieren, werden Kommentare ohne Überprüfung hinzugefügt.
  8. Enable Captcha (Captcha aktivieren) - aktiviert/deaktiviert Captcha unter dem Formular "Kommentar hinterlassen" auf der Produktseite.
  9. Enable Comment (Kommentare aktivieren) - aktiviert/deaktiviert Kundenkommentare.
  10. Show Author Name (Namen des Autors zeigen) -aktiviert/deaktiviert die Darstellung von Autor des Beitrags auf den Blogseiten
  11. Show Post Viewed (Angeschaute Beiträge zeigen) - aktiviert/deaktiviert die Darstellung von Beiträgen, die Benutzer schon gesehen haben. Dies befindet sich unter den Beiträgen auf den Bloseiten oder einzelnen Beitragsseite.
  12. Show Author Name Style (Format für den Namen des Autors) - ändert das Format für den Namen des Autors.
  13. AVATAR Image: (Foto:) - fügt das Standardfoto für alle Benutzer hinzu.
  14. Show No Image (Bild "Kein Foto" zeigen) - aktiviert/deaktiviert das Standardfoto. Dies wird für diejenigen Beiträge/Kategorien usw. benutzt, die kein Bild haben.
  15. Show Category (Kategorien zeigen) - aktiviert/deaktiviert die Darstellung von Bildern und Kategoriebeschreibungen auf den Kategorieseiten des Blogs.
  16. Blog Page Column Setting (Spaltenformat für Blogseiten) - Einstellung, die verbundene mit dem Blog Module darstellt. Sie enthält oft zusätzliche Einstellungen, deswegen ist es besser, die Standardmethode zu nutzen, um die zusätzlichen Module im Control Panel PrestaShop zu installieren.
  17. Custom CSS (Benutzerdefinierte CSS) - in diesem Feld kann man zusätzliche Stile hinzufügen. Nutzen Sie vorsichtig aktuelle Klassen des Themas, weil Probleme unter den Stilen entstehen können.
  18. Delete Old Thumblr (Altes Vorschaubild entfernen) - wird benutzt, um die Bilder des Blogs zu aktualisieren (nachdem die Bildauflösungen geändert oder eine neue Art hinzugefügt wurden). Um es hinzuzufügen, wählen Sie Yes (Ja) und drücken Sie Regenerate (Aktualisieren).
NUR DANACH können verbundene mit dem Blog Module installiert werden. Der Entfernungsprozess ist umgekehrt, WEIL ZUERST ALLE ZUSÄTZLICHEN MODULE und dann nur dieses Modul Smart Blog ENTFERNT WERDEN MÜSSEN.
RSS Feed Smart Blog

Das Modul erzeugt einen RSS-Feed für den Blog.

Es enthält einige Einstellungen im Control Panel:

  1. Update Period (Aktualisierungsintervall) - ermöglicht die Beiträge für eine bestimmte Zeitdauer (Stunde/Tag/Woche/Monat/Jahr) darzustellen.
  2. Update Frequency (Aktualisierungshäufigkeit) - ermöglicht die Aktualisierungshäufigkeit (1 Woche, 2 Stunden, usw.) festzulegen.

Oben befindet sich der Pfad zur Datei, wo die Nachrichten erzeugt werden. Sie besteht aus URL der Webseite + /modules/smartblogfeed/rss.php.

Archiv Smart Blog

Dieser Block wird in der linken/rechten Spalte dargestellt und sortiert die Beiträge nach dem Zeitraum (Monat).

Er hat keine zusätzlichen Einstellungen.

Kategorien Smart Blog

Dieser Block wird in der linken/rechten Spalte dargestellt und sortiert die Beiträge nach Kategorien.

Er hat keine zusätzlichen Einstellungen.

Neues auf der Hauptseite

Modul Media Parallax

Was ist eigenlich ein Parallax-Effekt (parallax)?

Parallax-Scrolling ist eine spezielle Scrolling-Technik in der Computergraphik. Dank diesem Effekt bewegen sich die Bilder im Hintergrund langsamer als im Vordergrund. Dadurch wird die Illusion der Tiefe und des zweidimensionalen Videospiel erstellt. Ab 2011 begannen Webdesigner das Parallax-Scrolling umzusetzen. Dafür haben sie HTML5 und CSS3 verwendet. Webseiten mit dem Parallax-Effekt sind gefragt geworden, weil er sie beweglich macht. Außerdem zieht der Parallax-Effekt scharfe Aufmerksamkeit der Kunden und macht jede Webseite einzigartig. Das Parallax-Scrolling verbessert das Aussehen der Webseite und ermöglicht dynamisches, interaktives Interface für Benutzer zu erstellen.

Dieses Modul fügt den Parallax-Effekt zur Webseite ohne zusätzliche Fähigkeiten und Mühe hinzu. Als Hintergrund können Sie Bilder oder Videos hinzufügen.

Einstellungen: (nur für Google Chrome)

  1. Enable smooth scroll for Google Chrome browser (Weiches Scrolling für den Browser Google Chrome hinzufügen) - zusätzlichen Effekt hinzufügen.
  2. Smooth scroll time (Zeit von weichem Scrolling) - Zeit von weichem Scrolling.
  3. Smooth scroll distance (Entfernung von weichem Scrolling) - Entferunung von weichem Scrolling.

Hinzufügung vom Modul Media parallax

  1. Es ist empfohlen, die Klasse oder den Index zum Inhalt der HTML Seite hinzuzufügen. Sie können auch einzigartigen Tag, der nicht wiederholt wird, nutzen.
  2. Erstellen Sie das Bild mit der notwendigen Größe (am besten ist, wenn das Bild größer als Größen des Blocks, zu dem der Effekt hinzugefügt wird, ist). Wenn Sie das Video verwenden möchten, müssen Sie es dann in drei Formaten: .mp4,.webm und .ogv vorbereiten und auch das Vorschaubild, das die gleiche Größe hat, (.png, .jpg und .gif) hinzufügen

  3. Klicken Sie auf den Button Add container (Container hinzufügen) in Add new block (Neuen Block hinzufügen).

  4. In dem DropDrownFenster legen Sie alle notwendigen Einstellungen fest.

    Einstellungen:

    1. Selector (Selector) - das Element, zu dem der Effekt (Index oder einzigartiger Tag) hinzugefügt wird.
    2. Parallax type (Parallax Art) - die Art des Effekts, die verwendet wird (Bild oder Video).
    3. Media files (Mediendateien) - laden Sie alle notwendigen Dateien hoch.
    4. Drücken Sie auf den Button Save (Speichern).

Cookie-Politik

Cookies sind kleine Textdateien, die die Webseiten auf den Computern der Besucher platzieren. Gewöhnlich werden sie zur Kundenidentifikation benutzt und sie erleichtern auch die Benutzung der Webseite. Cookies sind ein fester Bestandteil der Funktionen der Webseite, die wir immer nutzen.

Zum Beispiel, mittels Cookie speichern Online-Shops diejenigen Produkte, die Sie in den Warenkorb hinzugefügt haben. Cookies ermöglichen auch den Benutzern "im System" auf der Webseite zu bleiben und den Inhabern der Webseite die wichtige Information und Statistik über die Besucher zu bekommen.

Wenn Sie die Webseite haben, wird Cookie höchstwahrscheinlich auf dieser Webseite benutzt. Cookie wird meistens benutzt, um die Menge und Verhalten der Besucher zu folgen (indem man Werkzeuge Google Analytics nutzt). Man nutzt Cookies, um die aktuellen Angebote für die Besucher anzuzeigen. Cookies werden auch benutzt, damit die Werkzeuge des Online-Shops funktionieren können.

Am 26. Mai 2012 ICO Organisation hat die Änderungen zu den Rechtsvorschriften über Cookies veröffentlicht. Seitdem "diese Vereinbarung" erlaubt wurde, müssen Sie unbedingt zeigen, wo auf der Webseite Ihre Datenschutzrichtlinie und Cookies sind.

Zur Bequemlichkeit haben Sie das Modul Cookieslaw zum Ihrem Thema hinzugefügt.

Es kann einfach eingestellt werden.

  1. Finden Sie es in dem Ordner manual_install Ihrer Vorlage (wenn Sie die Vorlage manuell mit den Demodaten installiert haben, wird das Modul automatisch hinzugefügt).

  2. Dann finden Sie es in der Liste von Modulen im Control Panel (Modul Cookieslaw im Bereich "Funktionen front-office") und drücken Sie auf den Button Install (Installieren).

  3. Nach der Installation (überspringen Sie diesen Schritt, wenn Sie automatisch zur Seite der Moduleinstellungen gelangen sind) drücken Sie den Button Configure (Einstellen).

  4. Legen Sie alle notwendigen Einstellungen fest und drücken Sie auf den Button Save (Speichern).

    Einstellungen:
    • - Display at the top of the page(by default) (In dem oberen Teil der Seite darstellen (standardmäßig)). Wenn Sie diese Option aktivieren, wird das Modul in dem unteren Teil der Seite dargestellt.
    • - Cookies Policy link (Link zur Cookie-Politik).
    • - Link to the page where the user is redirected in case he does not accept the Cookies Policy (Link zur Seite, wo der Benutzer wird, falls er mit der Cookie-Politik nicht einverstanden ist).
